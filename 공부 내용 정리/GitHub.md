
## GitHub 이용해서 프로그램 연동하기

#### [git은 왜 사용하지?]
버전 관리가 가능하고, 협업할 때 편함.
변경사항까지 저장되서 수정 내용까지 확인 가능함.
>
---
#### [작동방식]
로컬(내컴퓨터)에 들어있는 파일을 원격(깃)에 올려줘야함. 
>
- git push: 나한테 있는 데이터를 원격(전체 메모리)저장소로 보내는 것.
- git pull: 원격저장소(전체 메모리)에 있는 데이터를 로컬저장소(내컴퓨터)로 가져오는 것

*만약 깃푸시 (업데이트 개념)를 안 해줄 경우에는 나만 갖고 있는 데이터임. 그리고 또 깃풀을 안 해줄 경우 공식적인 데이터는 원격임. 나만 없는 데이터임.

---

#### [생성 방법]
1. GitHub hwp 페이지에서 레포지토리 만들어주기
(레포지토리는 인스타에서 내게시물 같은 느낌, 내프로필 들어가서 생성 가능)
>
2. 로컬저장소(내컴퓨터)에서 파일 새로 만들기.
(내 파일 업데이트 준비)
>
3. VSCode 켜서 새로 만든 파일 켜기. 터미널에서 파일 추가해주기
**(파일 만들기 명령어): mkdir "파일명"**
**(폴더 만들기 명령어): touch 폴더명**
(이제 폴더를 만들거나, 다트파일을 만들거나 필요한거 만들어도 됨)
>
4. 터미널에서 **git init** 해주기 (깃 저장소 초기화 작업)
-> 이제 원격저장소(깃홈페이지)에 올릴 준비할거임. 로컬(내컴터)에 깃에서 관리할거라고 알려주는 작업.
>
5. 파일 저장하기. 명령어는 **git add .**
-> 모든 파일 다 저장하고, 변경사항 확인 후에 파일 올릴 수 있는 상태로 만들거임. 
*주의사항은 add 뒤에 . 붙이지말고 띄어쓰기 한 상태로 입력.
>
6. **git commit -m "올릴메세지"** 메세지와 함께 원격저장소(깃)에 메세지와 함께 올리는 작업. 
브랜치라는 곳에 기록됨.
>
7. 저장소 연결. (깃 홈페이지에 래포지토리 만들면, 생성되는 3줄 복붙)
-> 로컬이랑 원격이랑 연결시키는 작업
![alt text](image.png)
-> 대충 내용은 브랜치라는 곳에 여기 주인이고, 둘째 줄은 내 홈페이지는 여기야. 여기가 모든 저장소야. 마지막 줄은 깃푸시한다는거 저장한다는거.
>
그리고 끝. 내가 생성한 파일이나 수정한 내용들 홈페이지에 잘 올라갔나 확인하면 됨.

----

#### [생성된 후]
이제 로컬이랑 원격저장소 모두 잘 올라갔는지 확인 후에 만약 내가 VSCode에서 수정을 하거나 뭘 새롭게 생성했을 경우에 원격(깃홈페이지)에 꼭 다시 올려줘야함. 그게 최종 저장임.

**원격저장소에 올리는 법**
-> 참고로 처음했던 과정은 게시물 업데이트 과정이었다면, 지금 하는 과정은 올라간 게시물 수정 과정임.

1. **git add .** 
(파일 저장하는 과정)
>
2. **git commit -m "올릴메세지"**
-> 저 올릴메세지에 수정하고 메세지가 뜸
>
3. **git push** 
-> 내 로컬에 있는거 원격저장소로 민다. 가져가라.

> **주의사항**
매번 뭘 수정하거나, 새로 만들었을 때 파일이나 폴더가 저장되어있는지 꼭 확인. 파일명 옆에 흰색 점이 있을 경우는 저장 안 된 상태임.
>

---

#### [git 관련 용어]
-> 터미널에서 해당 글자만 치면 됨.

- **pwd** 지금 내가 작업하고 있는 파일의 경로 확인.
- **ls** 작업하고 있는 파일의 목록 보여주는 명령어.
- **cd** 이동시켜주는 명령어. ex) cd desktop
- **root** 최상위 경로로 이동 요청 명령어
- git status
현재 버전이나 파일 확인 명령어
- **git log --graph --online --decorate --<파일명>** 특정 파일에 대한 변경사항 올라갔는지만 확인하는 명령어

- **git log -- graph --online --decorate --all** 로컬 및 원격저장소에 모든 브랜치에 대한 커밋 내용을 표시해주는 명령어. 다 올라갔는지 그냥 확인해주는거.
