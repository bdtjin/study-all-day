# [Flutter 참고 사이트](https://docs.flutter.dev/ui/widgets/layout)

## MaterialApp
- 앱의 테마, 라우팅 등 설정하는 위젯
- 앱 내에서 최상위 위치, 보통은 하나만 사용

## Scaffold
- 페이지의 구조를 잡아주는 위젯
- 페이지당 하나만 사용
  - appbar, body, bottomNavigator or bottomSheet (body 영역은 항상 사용)

## AppBar
- Scaffold 상단의 고정크기 위젯
  - 자주 사용하는 곳은 이미 자동생성되어있음 (디자인 상으로 변경 가능)
  - leading: 뒤로가기
  - title: 보통 글자가 많이 들어감 
  *안드로이드에서는 기본적으로 왼쪽 배치 / 아이폰에서는 기본적으로 가운데 배치
  두개 다 동일하게 하고 싶다면? centerTitle: true 사용

## Text
- 문자열 출력하는 위젯
- Text("글자")
- `Text("글자", style: Textstyle(fontSize: 20, fontWeight: FontWeight.bold, height: 1))` -> 제일 자주 사용하는 스타일
  - `height`는 글자 높이 설정

## Image
- 이미지 출력하는 위젯
- `Image.asset('경로')` `Image.network('URL')`

## Mukti child 레이아웃 위젯
- Column 가로로 배치
- Row 세로로 배치
- Stack 자식 위젯들을 겹치게 배치 
  - Stack은 부모 위젯의 크기가 없을 경우, 제일 큰 자녀 위젯을 따른다 (제약 조건을 물려받는다.)
  - Positioned -> 컨트롤 가능
- ListView
  - 자식 위젯들을 세로로 정렬, 스크롤 가능한 위젯
  - children 속성에 Widget List 배치, builder 사용

## Single 
- Expanded 
  - 반응형 UI / 위젯을 꽉 채워줌 (부모 위젯)
- Container
  - 패딩, 마진, 꾸미기 색상, 테두리 등 크기 조정하는 위젯
- SizedBox
  - 자녀 위젯의 크기만 지정하는 위젯
- Center
- Padding 패딩을 설정하는 위젯
- AspectRatio: 특정 족황바로 크기 조정하는 위젯

---

- 화면에 보이는 모든 요소는 위젯이다
- 위젯은 클래스다. 상속받아서 빌드 메서드를 재정의를 해줘야한다.
- 먼저는 StatelessWidget으로 먼저 하고 -> 이후에 StatefulWidget으로 자동 변환 가능
- 레이아웃은 낼 아침스터디 시간에
